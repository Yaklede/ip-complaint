# Security & Compliance

> 본 문서는 제품 설계 시 고려해야 할 운영/증거/법무 경계 정리용이다.  
> 최종 법률 해석과 외부 제출 문서는 반드시 법무 검토를 거쳐야 한다.

## 1. 핵심 원칙

1. 외부 공인 IP의 실명 자동 특정은 제품 범위에서 제외한다.
2. 내부 조사 범위는 자사 보유 로그와 자산/계정 데이터에 한정한다.
3. 원본 로그와 증거 무결성 보존을 우선한다.
4. 자동 생성 문서는 초안이며 사람 승인이 필요하다.
5. 개인정보와 민감정보는 최소 표시/최소 보관 원칙을 따른다.

## 2. 제품이 할 수 있는 것과 할 수 없는 것

### 2.1 할 수 있는 것
- 내부 로그 상관분석
- 내부 단말/계정/사용자 귀속
- 외부 IP의 반복 패턴, ASN, 프록시/VPN 여부 정리
- 사건별 증거 패키지 생성
- 신고/고소장 초안 생성
- 법정 대응 체크리스트 제공

### 2.2 할 수 없는 것
- 외부 가입자 실명 조회
- 통신사/플랫폼 내부자료 직접 조회
- 수사기관 권한이 필요한 자료 자동 수집
- 자동 신고/자동 고소
- 법률상 결론 자동 확정

## 3. 신고/통지 관련 고려사항

### 3.1 개인정보 유출 통지/신고
공식 조문상, 개인정보처리자는 개인정보 유출 등을 알게 되었을 때 일정한 경우 정보주체 통지와 보호위원회 또는 전문기관 신고를 **72시간 이내**에 해야 한다.  
제품은 이를 자동 판정하지 않고, 사건 화면에서 체크리스트와 타이머를 제공하는 수준으로 설계한다.

### 3.2 침해사고 신고
정보통신서비스 제공자에게 해당하는 경우, 해킹 등 침해사고 발생 사실을 알게 된 때부터 **24시간 이내** 신고 요건이 문제될 수 있다.  
제품은 KISA/과기정통부 신고를 자동 제출하지 않고, 제출 준비 문서와 체크리스트만 제공한다.

## 4. 접속기록 및 감사로그 보관

개인정보처리시스템 접속기록은 일정 기준에 따라 **1년 이상**, 특정 경우 **2년 이상** 보관 요건이 있을 수 있다.  
따라서 제품은 다음을 충족해야 한다.

- 접속기록 보관기간 설정 가능
- 위변조 방지
- 반기 1회 이상 점검을 고려한 운영 점검 항목 제공
- 조사행위 자체에 대한 감사로그 저장

## 5. 통신자료/통신사실확인자료 경계

외부 사건에서 다음 정보는 제품이 임의로 취득할 수 없는 영역이다.

- 이용자 성명/주소/전화번호/주민등록번호 등 통신이용자정보
- 전기통신일시, 상대방 번호, 사용도수 등 통신사실확인자료

따라서 제품은 외부 공인 IP 사건에 대해 다음만 수행한다.

- 관측 사실 정리
- 반복성·시계열·패턴 정리
- 제출용 증거 번들 생성
- `추가 조회 필요` 상태 표기

## 6. 데이터 보호 요구사항

### 6.1 최소수집
- 문서 화면에서 주민번호 전체 표시 금지
- 사용자 연락처/이메일은 역할 기반 노출
- 원본 payload 표시 시 마스킹 전략 적용

### 6.2 접근통제
- 조사자, 법무, 개인정보 담당, 감사 역할 분리
- 문서 승인 권한과 사건 수정 권한 분리
- 증거 export 권한 제한

### 6.3 무결성
- raw artifact SHA-256
- generated document SHA-256
- export bundle manifest checksum
- 감사로그 append-only 성격 유지

## 7. 문서 출력 정책

문서에는 다음을 반드시 넣는다.

- 자동 생성 여부
- 생성 시각
- 사건번호
- 귀속 신뢰도
- 승인 상태
- 법무 검토 필요 여부

외부 사건은 다음 문구를 기본으로 한다.

- 피고소인: `성명불상(관측 공인IP 사용 행위자)`
- 비고: `통신사/플랫폼/수사기관 조회 필요`

## 8. 내부 운영 정책 권고

- 사건 문서는 법무 승인 전 외부 송부 금지
- 문서 버전 관리 필수
- 삭제보다 보존/접근통제를 우선
- 감사로그 무결성 점검 주기 수립
- 소스별 time sync 상태 모니터링

## 10. 구현 체크리스트

- [ ] 외부 실명 자동 기입 차단
- [ ] 자동 제출 기능 없음
- [ ] 원본/문서 해시 기록
- [ ] 문서 승인 흐름 존재
- [ ] 감사로그 존재
- [ ] 보존기간 설정 가능
- [ ] 민감정보 마스킹
- [ ] role-based access control
