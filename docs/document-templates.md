# Document Templates

## 1. 문서 생성 원칙

- 구조화된 사건 snapshot 기반
- 템플릿 렌더링
- 버전 관리
- 승인 전 DRAFT 표시
- 생성 입력값 보존

## 2. 사건 요약서 템플릿 필드

- 사건번호
- 사건명
- 작성일시
- 조사 담당
- 관측 IP
- 내부/외부 구분
- 귀속 신뢰도
- 핵심 사실
- 영향 범위
- 관련 시스템
- 관련 사용자/자산
- 증거 목록 요약
- 다음 조치

## 3. 증거목록 템플릿 필드

- 사건번호
- 증거 번호
- 증거 유형
- 원본 저장 위치
- 수집시각
- 해시(SHA-256)
- 관련 설명
- 관련 이벤트 ID

## 4. 고소장 초안 템플릿 필드

### 표지
- 사건번호
- 문서 버전
- 문서 상태
- 생성 시각
- 검토 상태

### 본문
- 고소인
- 피고소인
- 죄명(초안/검토 필요)
- 범죄사실 요지
- 발생 경위
- 피해 내용
- 증거 방법
- 수사 요청사항
- 첨부 증거 목록

## 5. 피고소인 표기 규칙

### 내부 사건 (A/B 등급)
- `홍길동 (사내 자산 LAP-00429 주사용자 / 귀속등급 A)`

### 내부 사건 (C 등급)
- `홍길동 외 1명 (공용 자산 또는 다중 사용자 환경, 추가 특정 필요)`

### 외부 사건 (D 등급)
- `성명불상 (관측 공인IP 203.0.113.10 사용 행위자, 추가 조회 필요)`

## 6. 템플릿 변수 예시

```json
{
  "document_type": "complaint_draft",
  "case_no": "INC-2026-000123",
  "generated_at": "2026-03-15T05:10:00Z",
  "actor": {
    "display_name": "성명불상",
    "confidence_grade": "D",
    "note": "통신사/플랫폼/수사기관 조회 필요"
  },
  "facts": [
    "2026-03-15 12:01~12:11 KST 동안 관리자 경로 반복 접근",
    "WAF 차단 로그 12건",
    "403 응답 188건"
  ],
  "evidence_manifest_id": "uuid"
}
```

## 7. 추천 구현 방식

- Jinja2로 HTML 초안 렌더링
- HTML -> PDF 변환 또는 DOCX 템플릿 렌더링
- 문서별 snapshot JSON 동시 저장
- 템플릿 버전별 폴더 분리

예:
```text
templates/
├─ case_summary/v1/
├─ evidence_manifest/v1/
└─ complaint_draft/v1/
```
